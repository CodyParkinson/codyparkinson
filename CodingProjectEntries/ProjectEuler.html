Project Euler

8/3/2022 - Current

Leonhard Euler

As of today (11/1/2022), I have solved the first 70 problems on ProjectEuler.net. It has taken me a few months, but I am excited to have completed the ones I have. Below I am listing my code for the first 100 problems. These are done in Python 3.9. Most of the problems run within the “one minute rule”, but some do not. Please read my notes to see how long to expect the programs to run. I have added notes where I feel they are applicable, but always feel free to reach out if something doesn’t make sense.

As for the difficulty, I made notes in the code about how long it took me to solve each problem. I did all of the problems in order; I did not skip around. All of the code was written by me, unless specifically stated otherwise. When researching better methods for solving parts of problems, I copied techniques that were optimal; again, this is all noted. Finally, it really is hard to gauge how difficult a problem is based on its percentage. There were quite a few 25% problems that I felt were easier than the 5% and 10% ones, so when attempting these yourself, don’t focus on that number.

This leads me into thinking about tips for solving. Again, don’t focus on the difficulty, this really doesn’t mean very much (at least on the ones below 50%). All of the problems have a trick or a mathematical equation to them. Do not try to hard code and iterate through all the values a problem could have. Most problems deal with such a large amount of values that it is impossible to solve without the proper equations. Research, research, research - this is not cheating. Looking up answers is cheating, but use Google (not Bing) to find hints in solving. Finally, make sure your program can solve for the examples. If the problem gives examples, make sure you can solve them. So many times I jumped in, wrote a bunch of code, and realized I couldn’t even solve for the first (usually low) values.

And that’s about it! I love solving these, the feeling of dopamine you get after grinding out a problem for hours and days is hard to match. Please read my note at the bottom of the page, but other than that, happy Eulering!
